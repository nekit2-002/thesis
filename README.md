# Thesis
Пояснения для тех, кто задумает воспользоваться этим шаблоном, написанном на typst -- 
инструменте для верстки, о котором я недавно узнал.

## Устройство
Данный шаблон разделен на несколько основных частей:
- main.typ -- главный файл проекта, от которого начинаем плясать; он содержит настройки,
  глобальные для всей ПЗ (типа полей на страницах, общего шрифта, настроек переноса)
- preamble.typ -- файл с определением всяких вспомогательных настроек и переменных
  по типу темы, названия работы и тд, а также некоторых функций для облегчения жизни
- contents.typ -- файл, отвечающий за структуру главной части пояснительной записки
  (то есть всего, что лежит вне титульника и задания); содержит
  инклюды содержимого всех частей ПЗ плюс некоторые настройки, которые характерны только для отдельных из них
- директория title -- набор файлов, связанных с версткой титульника, который при желании можно
  найти на сайте кафедры, но я решил сделать его вручную, чтобы можно было отредактировать все
  что хочу на лету
- директория task -- набор файлов связанных с версткой задания. В данный момент разделен на
  две части: task.typ -- основной файл с версткой титульной части задания, и task_table.typ
  -- файл с версткой таблицы с заданиями по пунктам
- директория chapters -- набор файлов, содержащих основные разделы записка; также содержит
  файл со ссылками на источники biblio.bib и .csl файл с гостом для списка литературы
  (чтобы можно было подогнать всякие мелочи если вдруг что)
- директория appendices -- набор файлов с приложениями кода для ПЗ
- директории listings и images -- для вставляемых в записку листингов и
  изображений соответственно

## Сборка
Для сборки необходим единственный компонент -- **компилятор языка typst**,
имеющий одноименное имя. Устанавливается на mac просто:
```
$ brew install typst
```

После установки компилятора можно собрать записку двумя способами:
либо вручную запустить

```
$ typst compile main.typ main.pdf
```
либо же воспользоваться Makefile и выполнить
```
$ make
```

Для очистки от артефактов сборки можно использовать
```
$ make clean
```
